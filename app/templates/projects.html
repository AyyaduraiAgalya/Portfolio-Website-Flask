Tools & Technologies

  • API: OANDA API
  • Project Management: Jira, GitHub

Fraud Detection System

Timeline: Completed

Brief Description: This project leverages distributed data processing and machine learning to detect fraudulent transactions in credit card data. It combines end-to-end data science workflows with scalable tools like PySpark to simulate real-world data handling and predictive modeling.

Key Responsibilities & Contributions

  • Data Pipeline Development: Built a scalable data pipeline using PySpark for loading, cleaning, and processing credit card transaction data.
  • Exploratory Data Analysis: Conducted EDA to understand class imbalances, distribution patterns, and potential fraud indicators.
  • Feature Engineering: Engineered fraud-related features, including transaction frequency, average transaction amounts, and rolling standard deviations within 5-min and 1-hour windows.
  • Preprocessing: Standardized numerical features and split data into train/test sets, ensuring consistency in model inputs.
  • Model Development: Trained and evaluated a logistic regression model using PySpark MLlib, achieving high AUC and F1 scores.
  • Evaluation Metrics: Assessed model performance using advanced metrics like Precision, Recall, F1 Score, ROC-AUC, and False Positive Rate.

Tools & Technologies

  • Distributed Processing: PySpark
  • Data Manipulation: Pandas, NumPy
  • Data Visualization: Matplotlib
  • Machine Learning: Logistic Regression with PySpark MLlib

Impact & Outcome

  • Achieved a high AUC score of 0.9709, demonstrating strong model performance in distinguishing fraudulent and legitimate transactions.
  • Delivered a scalable fraud detection solution, simulating real-world data processing and machine learning workflows.
  • Enhanced the project's analytical depth with engineered features that significantly improved detection sensitivity.

Future Work
